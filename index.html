    Scalable IoT Platform

    IOT Research Center, Amirkabir University of Technology, Tehran, Iran
    Sep 2017 - Sep 2018
    Supervisor: Prof. Bahador Bakhshi

    Amirkabir Scalable IoT Platform (ASITP) has been designed and developed by Amirkabir University IoT Group. This project is a Docker-based platform that used in IoT middleware layer. Currently, a government company is using the platform for smart agriculture applications. The platform has 4 layers: Main Backend, App Backend, App frontend and container Orchestration. The main future of the platform is supporting enormous size and heterogeneous devices.


    Today sensors and actuators that communicate on the LoRaWAN can connect to ASITP. The communicate data collected by LoRa Server. The communicate data collected by LoRa Server. This server also handles Device configuration ‚Äì like encryption methods or data sensing period ‚Äì and gateway setup.



    figure 1: Platform Architecture- Designed by prof, Bahador Bakhshi


    The platform has the following functions:
         ‚Ä¢ Register, authentic and authorise users
         ‚Ä¢ Setup LoRa gateway
         ‚Ä¢ Add and configure IoT devices ‚Äì sensor and actuator.
         ‚Ä¢ Activate IoT devices ‚Äì ABP and OTAA.
         ‚Ä¢ Use user-defined encoder and decoder ‚Äì in a scripting language - for data encryption and decryption.
         ‚Ä¢ Report sensed data in tabular and chart version
         ‚Ä¢ Send data to actuators
         ‚Ä¢ Execute user-defined scenarios ‚Äì in a scripting language.



    figure 2: Simple report

    The second layer is App Backend, the layer implemented by PHP on Laravel framework. The functions of this layer are the management of users, roles and payments and connect UI to Main Backend. This layer also saves user data, logs and payment receipts to MongoDB.



    figure 3: Platform UI


    The App Frontend ‚Äì or user interface ‚Äì is developed by NodeJs on the React framework. The users use this interface to login and setup IoT device. This layer presents the sensed data by a table and different charts.


    In this platform, we have used the containers for execution user-defined scripts. Container Orchestration layer manages the Dockers. Kubernetes is the solution!



    figure 4: Sample script-based scenario


    The brilliant idea of this project is using user-defined scripts - as decoder, encoder and scenario. Because of this feature, the platform supports heterogeneous devices.


    Upcoming functionalities of the platform are:
         ‚Ä¢support data analysis tools
         ‚Ä¢support mobile applications
         ‚Ä¢support MBIoT network



    figure 5: Future developments- Designed by prof, Bahador Bakhshi
